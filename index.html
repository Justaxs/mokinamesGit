Git

Git'as yra populiariausia kodo versijavimo programa.

Git komandos

  • init - sukuria git'o repozitoriją dabartiniame aplankale
  • add - prideda nurodytus failus prie commit'ui paruoštų failų (stage'ina)
  • commit -m "žinutė" - sukuria kodo versiją iš stage'intų failų ir prideda privalomą žinutę apie pakitimus (žinutė turėtų būti neilgesnė nei 50 simbolių)
  • status - nurodo informaciją apie tavo lokalios git'o repozitorijos statusą
  • push - iš lokalios repozitorijos iššsiunčia commit'us į išorinę repozitoriją
  • clone - klonuoja nurodytą repozitoriją iš išorinės repozitorijos.
  • log - grąžina commit'ų sąrašą
  • chechout branchName|commitID - pereinama į nurodytą šaką arba kodo versiją.
  • branch naujosSakosPavadinimas - sukuria naują šaką.

branch protection rules

Viena svarbiausių šakų apsaugos taisyklių yra Pull request nustatymas.

Eiliškumas Pull Request

  1. Susikuriama nauja šaka - git branch newBranchName
  2. Nueinama į naują šaką - git chechout newBranchName
  3. Atlieki užduotį, kurią apsiėmei atlikti toje šakoje
  4. Add'inai ir commit'inai atliktus veiksmus (nebūtinai vienas commit'as) - git add . IR git commit -m 'message'
  5. Atnaujinti main šaką - git pull
  6. Nueinama į naują šaką - git chechout newBranchName
  7. Sujungi savo šaką su main ir išsprendi galimus konfliktus - git merge main
  8. Nusiūsti naujos šakos pakitimus į gitHub'ą - git push
